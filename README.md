# Nombre del Proyecto: Examen 1

## Problemática: Falta de portafolio digital 

Muchos estudiantes no cuentan con una presencia de forma digital, que les permita mostrar sus trabajos, ni
generar un portafolio profesional; lo cual, puede interferir en su proceso de conseguir trabajo.

## Objetivo: 

Nuestra pagina busca darles a esos estudiantes un lugar donde puedan hablar de si mismos y sus habilidades,
ademas de mostrar evidencias de proyectos previos que muestren de lo que son capaces. Con esta herramienta,
esperamos que se les facilite generar un portafolio profesional digital, y les ayude a encontrar trabajo.

## Conflicto generado 

El conflicto ocurrió porque dos integrantes modificaron el mismo párrafo en ramas diferentes. 
Este conflicto se le conoce como conflicto de fusion (merge conflict)

## Cómo lo resolvimos 

 Debido a que el programa no puede editar el codigo, ya que no sabe que conservar, nos toca a nosotros elegir de forma manual el archivo conflictivo para seleccionar los cambios que quieres mantener en la fusión final. Esto lo podemos realizar desde el editor de conflictos desde GitHub.
 Como dato, al solucionar este tipo de problemas, toda la rama base de la solicitud de cambios se combina en la rama principal activa; por lo tanto, hay que tener cuidado a la hora de cuidar que fusionamos y que conservamos, pues todo se juntara a la rama activa.

## Qué aprendimos 

Aprendimos que este tipo de problemas pueden ser algo dificiles de solucionar, pero con algo de buena comunicacion entre miembros del equipo, es posible evitar completamente estas situaciones; incluso a la hora de resolver estos problemas, hablar entre el equipo de trabajo puede ayudar a decidir que se va a conservar y que se va a borrar en el repositorio nuevo.
